Introduction

There are many programming languages for writing dynamic yet safe code. You might already have a favorite one! However, safety often comes at the cost of reduced expressiveness, like mutability restrictions or relies on non-infallible programers. Blombly is a language that aims to address these issues with a small base instruction set that is easy to learn but promises code and memory safety by design. It also focuses on algorithmic logic by automating tasks like memory management and parallelization without losing expressiveness. All its interfaces further follow an "one solution" principle, meaning that there should be one obvious way to implement code in most situations. For example, there is only one loop structure (while) and alternatives like iterating over all list elements are implemented as macros that use it.

Blombly compiles into an intermediate representation that is interpreted by the BLOck aseMBLY Virtual Machine (blomblyVM). The latter ensures reliability by replacing jump-based statements to entering and exiting code blocks. This includes managing control flow via conditions, loops, inlining, method calling, return statements for successful operations, and error messages for failures.

So, who is Blombly for? It might be a good choice if you want to:

✅ Quickly learn a dynamic scripting language.
✅ Write sequential code that runs in parallel.
✅ Ensure memory and logic safety by design.
