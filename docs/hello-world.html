Hello world!

Let us create a first program to show a compilation and interpretation process. Create the following example file, where the .bb extension is associated with blombly source code. Careful to not forget the trailing semicolon, as this would create an error when trying to compile and execute the file. In general, there is enforcement of unambiguous code syntax that prevents code smells and has a unique way of implementing every functionality. Use a Java syntax highlighter (but not a syntax checker), as the two languages share many keywords.

main.bb 
print("Hello world!");

To run this file, download the blombly executable from the language's GitHub page and add it to the filepath. Then run the following console command:

> blombly main.bb
( OK ) Compilation
( OK ) Optimization
Hello world!

Success! If a message that starts with ( ERROR ) appeared, the language was set up properly, but there was some syntax or logic error: there may be a missing file or semicolon, usage of the wrong type of brackets, or some other infraction. In this case, read the error's details carefully, as they are curated to be highly descriptive. More information on errors is provided in the next section. If another type of console error message appeared, please make sure that the blombly executable is compatible with your operating system and properly added in the file path.

Compilation creates an abstract syntax tree and converts it into the blomlyVM intermediate representation language, shown below, which can also run directly. BlomblyVM files always have the .bbvm extension and look like assembly code. Temporary variables starting with _bb are also created. Often, compilation also optimizes the code to take advantage of language features like caching memory locations for result outcomes and declaring temporary code blocks as final to avoid redefinition during loops. As a final remark, blomblyVM files pack all necessary dependencies internally, and can therefore be shared for direct execution.

main.bbvm 
BUILTIN _bb0 "Hello world!"
print # _bb0

The script bbreader.py (written in Python) converts compiled code to the main language for easier reading, although without removing the compilation's intemediate variables.
