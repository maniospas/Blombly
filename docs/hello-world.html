Hello world!

Let's make a first program to demonstrate the compilation and interpretation process. Create the following example file, where the .bb extension is associated with blombly source code. Do not forget the trailing semicolon, as error messages are created for wrong syntax or code smells that would be hard to debug. Use a Java syntax highlighter (but not a syntax checker), as the two languages share many keywords.

main.bb 
print("Hello world!");

To run this file, download the blombly executable from the language's GitHub page and add it to the filepath. Then run the following console command:

> blombly main.bb
Hello world!

Success! If a message that starts with ( ERROR ) appears, the language was set up properly but there was some syntax or logic error. For example, brackets may have closed with a semicolon, or there could be some other type of infraction. Read the error details carefully, as they are curated to be highly descriptive. More information on errors is provided in the next section. If another type of console error message appears, make sure that the blombly executable is compatible with your operating system and properly added to the file path.

Compilation creates an abstract syntax tree and converts it into the blomlyVM intermediate representation language, like the one shown below. BlomblyVM files are associated with the .bbvm extension and look like assembly code. They pack any dependencies internally and be run directly; thus they can be shared directly for execution by others. Concerning their contents, temporary variables starting with _bb are created. Compilation also optimizes the code to take advantage of virtual machine features like cached memory locations and declaring temporary code blocks as final to avoid redefinition during loops.

main.bbvm 
BUILTIN _bb0 "Hello world!"
print # _bb0
